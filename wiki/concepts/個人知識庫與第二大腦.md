# 個人知識庫（第二大腦）

個人知識庫是將個人過去累積的專業內容（講義、文章、課程、答疑紀錄等）整合成一個結構化的資料夾，讓 Claude 可以查詢並產出符合個人風格與觀點的內容，而非四平八穩的通用回答。

> **白話說明**：把你過去寫的東西、學到的東西整理成一個資料夾給 Claude 讀，這樣 Claude 就能以「你的口吻和觀點」來回答問題，而不是說一堆沒有個人特色的廢話。

## 重點

- 知識庫是 AI 輸出「個人化內容」的基礎，取代通用知識
- 內容可包含：專業講義、部落格文章索引、過去課程影片逐字稿、客戶問答紀錄
- Claude 讀取知識庫後進行語義比對，找出最相關的內容
- 可搭配[[concepts/Skill技能手冊]]建立自動化查詢流程
- 知識庫中的部落格文章索引可讓 Claude 自動判斷並置入相關連結

## 應用場景

- 以個人教學觀念回覆論壇問題
- 家教面談前自動整合學員資料
- 客戶跟進流程自動化

## Karpathy 方法：最簡單的實作方式

Andrej Karpathy 提出：不需 RAG 或 vector database，只需一個本地資料夾 + Claude Code 就能建立可查詢的 AI 知識庫。將知識結構化整理後，Claude 不需每次讀取全部原始內容，可節省最高 95% 的 token 用量。本知識庫（Max KnowledgeBase）即是這套架構的實踐。
